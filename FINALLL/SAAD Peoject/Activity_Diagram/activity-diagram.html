Full description

This flowchart illustrates the complete path a student follows when using the course management system, beginning from entering the system and checking whether they are already registered. If the student is not registered, they are directed to fill out a registration form which is validated by the system, and they cannot continue until the form is completed correctly. If the student is already registered, they move directly to logging in. After successful login, the student reaches the course management area, where the system checks whether a course has already been created. If no course exists, the student is required to create one before continuing; otherwise, they may update existing course information. The student then proceeds to search for courses either by name or by credit hours. If no course matches the search criteria, the system displays an error and allows the student to try again. If a course is found, the student selects it and completes the process by exchanging their current course with the chosen one, which marks the end of the workflow.

Detailed breakdown

  • Start & End: Trigger events and termination states (normal / error).
  • Decision nodes: Conditional branching with guard labels (e.g., validationPass / validationFail).
  • Fork / Join: Parallel tasks (e.g., notifications + analytics) and synchronization points.

How to read & tips

  1. Follow arrows from the filled start circle downward.
  2. At diamonds, follow guard labels to determine the path.
  3. Convert decision branches into focused test cases (both positive and negative).
  4. Annotate actions with expected duration, retries and failure handling for implementation clarity.

Practical advice

Label important nodes and actions with IDs or brief notes to make cross-referencing with requirements and test cases easier. For long diagrams, include a small legend or numbered references.
