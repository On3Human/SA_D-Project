University Management System Class Diagram

Description of the Class Diagram

This class diagram illustrates the structure of a University Management System. It defines several key entities and their relationships:

  • User: The base class with common attributes like ID, name, email, password, phone, and address. It includes methods for login, logout, and profile updates.
  • Student: Inherits from User and includes specific attributes like student level and GPA. It has methods to register for courses, view grades, and download material.
  • Professor: Also inherits from User, with additional attributes for salary and degree. Professors can upload grades, upload material, and manage courses.
  • Admin: Inherits from User and has a 'role' attribute. Admins have system-level management capabilities, including adding students, adding professors, and managing the overall system.
  • Department: Represents a university department with a department ID and name. It can retrieve a list of courses offered by the department.
  • Course: Defines course details such as course code, name, and credit hours. It includes methods to get course details and set the course schedule.

Relationships depicted in the diagram:

  • A Department contains multiple Students and employs multiple Professors.
  • A Department offers multiple Courses.
  • A Student registers for multiple Courses.
  • A Professor teaches multiple Courses.
  • Student, Professor, and Admin all inherit from the User class, indicating common functionalities and attributes.
