High-level view of the University Management System and its external actors

Home · Level 1 · Prototype

Humanized description

The context DFD presents the University Management System (the central bubble) and two primary external entities: Student and Admin. Key flows include:

  • From Student → System: Student ID, personal info, course selection, payment info (inputs)
  • From System → Student: Class schedules, grade transcripts, fee receipts, alerts (outputs)
  • From Admin ↔ System: Curriculum/policy updates, enrollment reports, system logs, and administrative data

Why it matters

This level clarifies boundaries and what data crosses them — useful for security, integration and stakeholder agreements.
