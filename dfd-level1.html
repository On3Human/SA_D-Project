Level 1 DFD — High-Level System Processes

Breakdown into core processes and data stores

Home · Context · Prototype

Processes explained

  • 1.0 Manage Admissions & Profile: handles student onboarding, profile updates and admission confirmations. Data store: Student DB (D1).
  • 2.0 Manage Academics & Registration: course catalogs, enrollment, class timetables. Data store: Course DB (D2).
  • 3.0 Manage Examinations & Grades: exams, grade entry and transcripts. Data store: Grade DB (D3).
  • 4.0 Manage Fees & Finance: billing, payment verification and receipts. Data store: Finance DB (D4).

Interactions & flows

Student and Admin exchange application details, enrollment forms, grade reports and fee confirmations with the system; backend data stores hold authoritative records used by multiple processes.
