Seeded accounts

  • student / studentpass
  • instructor / instructorpass
  • admin / adminpass
  • finance / financepass (Finance Dept)

Key behaviors

  • Student: register up to 7 courses; below 4 courses shows a warning.
  • Registration includes payment simulation; Visa may fail randomly; Cash always succeeds. If Visa fails, Finance can approve pending payments.
  • Course codes are unique ignoring case; editing a code will update the existing entry.
  • Exchange flow attempts an atomic drop+add and prevents exchanging to a course already taken.

Troubleshooting

  1. If navigation redirects to login, sign back in.
  2. If payment fails, choose Cash or wait for Admin/Finance to verify pending payments.
  3. If a course disappears after delete, it also removes associated registrations and payments.
