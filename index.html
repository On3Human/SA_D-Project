University Management — Front-end Prototype

Fast prototype implementing the diagrams: course catalog, registration with payment flow, finance verification, grade management and admin reporting. Data persists in localStorage.

Auth & Roles

Student, Instructor, Admin, Finance — role-based flows and UI.

Registration

Seat checks, payment stub, pending payments and finance verification.

Course Management

Create / update / delete courses with credits and seat counts.

Seeded accounts

  • student / studentpass
  • instructor / instructorpass
  • admin / adminpass
  • finance / financepass
Tip: use Cash for deterministic payment success during testing; choose Visa to generate pending payments for finance to approve.
Prototype — static front-end only. Data stored in localStorage (ums_data_v3). To reset demo state remove the key in DevTools → Application → Local Storage.
