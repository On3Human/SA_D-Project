Sequence Flow — Course Registration

Narrative of the sequence diagram (login → select → seat check → payment → confirm)

Activity · Prototype

Step-by-step narrative

  1. Student logs in — system authenticates (token/session).
  2. Student requests course list; system displays available courses.
  3. Student selects course and initiates registration — system checks seat availability with registrar service (simulated).
  4. If seats available: proceed to payment verification with Finance; on success, enrollment recorded; on failure, ask user to retry.
  5. System returns confirmation to student and updates course seat counts and records.

The prototype simulates these steps on the Prototype page.
