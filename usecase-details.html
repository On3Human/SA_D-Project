Login

Pre: user has an account; Post: user is authenticated (mock token in client).

  1. User supplies username/password.
  2. System validates credentials (mock check).
  3. On success, user lands on the prototype dashboard; otherwise error shown.

Register for a Course

Pre: user is authenticated; Post: student registered and seat count decremented.

  1. Student selects a course from search results.
  2. System checks seat availability (simulated).
  3. If seats available, system requests payment verification (simulated finance).
  4. On payment ok, registration confirmed. Otherwise, user prompted to retry.

Manage Courses (Registrar)

Registrar can create/update/delete courses; changes affect registration availability in prototype.
