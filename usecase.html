Actors and top use cases

  • Student: login, view available courses, register for courses, view grades, pay fees.
  • Instructor/Registrar: manage courses, view student list, enter grades, publish grades.
  • Admin/Finance: run reports, verify payments, update records.

The use case diagram informs the UI screens in the prototype (authentication, course management, registration, payments, grades and reports).
